Chapter1-从源码编译构建比特币核心节点

最近在实验课中接触到了这方面的相关教程，但我发现我们老师所发的教程手册中有很多没有提到的内容，所以在这里开了一篇文章来详细介绍一下

其实在实际操作中遇到最多的是报错/运行不能/网络错误等问题，这些我看教程里都没有提到，所以我个人进行了一些简单的补充，主要内容还是以原教程为主

并且由于这部分的内容较多，我将分成几个部分来讲，这篇文章先讲述如何从源码编译构建比特币核心节点

1. 云服务器的获取

这一步就没什么好说的，随便找个华为云/腾讯云/阿里云的服务器就行，可以用免费试用（毕竟只是测试使用），注意云服务器的系统使用linux的ubuntn

2. 构建Bitcoin Core核心节点

(1). 使用git命令行在系统上创建Bitcoin Core源代码的本地副本

先使用 sudo apt-get install git 命令在虚拟机上安装git，再git clone一下bitcoin的github仓库地址

这一步经常出现网络差（github通病）拉取失败的报错，可以在github.com前再加上一个gitclone.com解决

(2).使用git命令查看并checkout对应的Bitcoin Core版本**

克隆仓库完成后进入bitcoin文件夹，可以使用 git tag命令 查看比特币版本信息

我们这里选择v23.0版本进行后续操作

通过命令git status来确认当前源代码的版本信息

(3).配置构建Bitcoin Core系统

在Ubuntu Linux中配置与编译Bitcoin Core系统前，可以使用apt安装所需的依赖库。

安装途中会不断弹出是否要继续安装的选项，一直输入Y（yes）就行

使用./autogen.sh脚本生成一组build 脚本来启动build过程

以下是一些常用的选项，可以覆盖configure脚本的默认行为：

–prefix=$HOME 这将覆盖生成的可执行文件的默认安装位置（Linux系统默认是/ usr / local /）。 可以使用$ HOME将所有内容放在用户的主目录或不同的路径中。

–enable-wallet 用于启用钱包功能。

–disable-wallet 用于禁用钱包功能。

–without-bdb 钱包不使用已不推荐使用的Berkeley DB数据库。

–with-sqlite 钱包使用SQLite数据库。

–with-gui=no 不要构建图形用户界面，图形界面需要Qt库，只构建服务器和命令行。

输入下面这串命令确认依赖已经全部安装成功

*如果一切顺利，configure命令将会以创建可定制的构建脚本结束。这些构建脚本允许编译bitcoind。如果有缺失的库或是错误，configure命令将会以错误信息终止。如果出现了错误，可能是因为缺少库或是有不兼容的库。 重新检查构建文档，确认你已经安装缺失的必备条件，然后运行configure，看看错误是否消失了。*

(4).构建Bitcoin Core系统可执行文件

这一步将编译源代码，这个过程根据CPU和内存资源不同，但一般可能需要1个小时完成。在编译的过程中，应该过几秒或是几分钟看一下输出结果。如果出现了问题，会看到错误信息。如果中断了，编译的过程可以在任何时候恢复。

这一步确实要编译很久，根据网络和虚拟机配置不同有较大的差异，耐心等待即可

上一步完成后，通过sudo make install 命令，安装 bitcoind 可执行文件到Linux系统路径下

最后再检查一下bitcoin是否安装成功

出现这样的提示就是成功了
